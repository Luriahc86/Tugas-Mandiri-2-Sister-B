Penjelasan Singkat
Request-Response — Client mengirim permintaan dan menunggu balasan server secara sinkron. Jika server tidak merespons dalam batas waktu → terjadi timeout. Cocok untuk: HTTP REST API, SQL query, RPC.

Publish-Subscribe — Publisher kirim pesan ke topic melalui Broker. Subscriber yang terdaftar ke topic itu otomatis menerima pesan (asinkron, loosely-coupled). Cocok untuk: MQTT, Apache Kafka, Redis Pub/Sub, notifikasi IoT.
